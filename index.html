What's inside

One platform, every data type.

Each project picks an annotation module tailored to its data — physiological signals, free-text notes, extracted entities, or model outputs.

ecg_heart

AFib Waveform

Multi-lead ECG viewer with zoom & pan. Label segments as AFib, normal, or noisy — assisted by a detection model and fine-tuning.

monitoring

Signal Quality (SQi)

Drag to select time ranges across stacked PPG/ECG channels and grade signal quality region-by-region.

description

Clinical Notes

Review highlighted concepts in real clinical notes and answer structured, conditional questions per concept.

label

Entity (NER)

Validate extracted entities — drugs, persons, conditions — tagged inline over the source text from MetaMap/NLP pipelines.

smart_toy

LLM Output Eval

Score model answers on factual accuracy, completeness, logic, and detection with rating scales — build LLM evaluation datasets.

Navigate studies and series in a medical image viewer, then apply classification labels frame-by-frame — add new labels on the fly.

Multimodal Cohorts

03 — Signal Quality

Grade by region.

Select one or many time windows across stacked channels — highlighted in place — and submit a quality grade for each. Hover any point for its exact value and timestamp.

04 — Clinical Notes

Concept-level review.

Highlighted concepts in the note drive a structured question form on the right — is the concept correct, in the right encounter, and what's its status? Step through every concept, one of N.

05 — Entity Annotation

Validate extractions.

Inline PERSON and DRUG tags from the NLP pipeline, shown over the raw note for fast human verification and correction.

06 — LLM Evaluation

Score the model.

Rate model-generated answers across multiple axes — detection, factual accuracy, completeness, logic, and encounter grounding — to build rich evaluation datasets for clinical LLMs.

07 — Adjudication

Resolve to gold standard.

The adjudicator sees every annotator's answer and inter-rater agreement for each item, then picks the final value. The same note + question surface, now with consensus baked in.

How it works

From cohort to gold standard in four steps.

01

Cohort

An admin creates a project, attaches a form, and loads the cohort's files and events from the bucket.

02

Assign

Users are assigned as annotators or adjudicators, and events are distributed for review.

03

Annotate

Annotators label their assigned events in the module that fits the data — waveform, text, NER, or LLM eval.

04

Adjudicate

Adjudicators reconcile multiple annotators into a single gold-standard value, ready for export and reporting.

Access control

Three roles, cleanly separated.

A global app role controls the shell; a per-project role controls the job. One role per user per project keeps annotation and adjudication independent.

admin_panel_settings

Admin / Manager

Creates projects & forms, manages users, assigns roles and files, and views reports. App-level role.

edit_note

Annotator

Labels assigned events within their projects. Works independently — can't see other annotators' answers.

gavel

Adjudicator

Reviews all annotators' answers per event and sets the final, gold-standard value. Never adjudicates their own annotations.
